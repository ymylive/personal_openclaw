# P1: 下载引擎与解压模块

> 前置依赖: P0（共享类型定义）
> 可并行: 与 P2、P4 并行开发
> 预计工作量: 2天
> 产出物: downloader.rs + extractor.rs — 可从URL下载任意文件并解压zip/7z/tar.gz

---

## 一、项目背景（简要）

我们在开发一个 Rust TUI 安装器，自动部署 VCP 系统。本阶段负责**通用下载引擎**和**通用解压引擎**——后续阶段（P2 Portable运行时、P3 部署引擎）都依赖这两个模块。

### 你需要了解的架构

```
用户双击exe
  → TUI界面收集配置
  → 启动后台安装任务（tokio::spawn）
  → 后台任务通过 mpsc::Sender<ProgressEvent> 向TUI发送进度
  → TUI实时刷新显示
```

P1的下载器和解压器就是"后台安装任务"中最底层的工具函数。

### 4.1 公开接口

```rust
use std::path::{Path, PathBuf};
use tokio::sync::mpsc;
use crate::app::ProgressEvent;

/// 下载配置
pub struct DownloadConfig {
    /// 下载URL
    pub url: String,
    /// 保存到的本地路径
    pub dest: PathBuf,
    /// 对应的安装步骤索引（用于进度回报）

/// - 流式下载（边下边写，不占大内存）
/// - 实时进度回报（通过progress_tx）

/// - 超时重试（3次）
pub async fn download_file(
    config: DownloadConfig,
    progress_tx: mpsc::Sender<ProgressEvent>,

/// 从GitHub API获取最新Release的下载URL
/// 用于获取PortableGit、NewAPI等的最新版本
pub async fn get_github_release_url(
    repo: &str,          // 如 "git-for-windows/git"
    asset_pattern: &str,  // 如 "PortableGit" 和 "64-bit.7z.exe"

/// 获取Node.js LTS最新版本号
pub async fn get_nodejs_lts_version() -> Result<String>;
```

### 4.2 实现要点

#### 流式下载核心逻辑
```rust
use reqwest::Client;
use futures_util::StreamExt;
use tokio::io::AsyncWriteExt;

pub async fn download_file(
    config: DownloadConfig,
    progress_tx: mpsc::Sender<ProgressEvent>,
) -> Result<PathBuf> {
    let client = Client::builder()
        .timeout(std::time::Duration::from_secs(300))  // 5分钟超时
        .build()?;

    // 断点续传：检查已有文件大小
    let mut start_byte: u64 = 0;
    if config.resume && config.dest.exists() {
        start_byte = tokio::fs::metadata(&config.dest).await?.len();
    }

    // 构建请求
    let mut request = client.get(&config.url);
    if start_byte > 0 {
        request = request.header("Range", format!("bytes={}-", start_byte));
    }

    let response = request.send().await?;

    // 检查状态码
    if !response.status().is_success() && response.status().as_u16() != 206 {
        anyhow::bail!("下载失败: HTTP {}", response.status());
    }

    // 获取总大小
    let total_size = if response.status().as_u16() == 206 {
        // 断点续传响应，从Content-Range解析总大小
        response.headers()
            .get("content-range")
            .and_then(|v| v.to_str().ok())
            .and_then(|s| s.split('/').last())
            .and_then(|s| s.parse::<u64>().ok())
            .unwrap_or(0)
    } else {
        response.content_length().unwrap_or(0)
    };

    // 打开文件（追加模式 or 新建）
    let mut file = if start_byte > 0 {
        tokio::fs::OpenOptions::new()
            .append(true)
            .open(&config.dest)
            .await?
    } else {
        // 确保父目录存在
        if let Some(parent) = config.dest.parent() {
            tokio::fs::create_dir_all(parent).await?;
        }
        tokio::fs::File::create(&config.dest).await?
    };

    // 流式下载
    let mut stream = response.bytes_stream();
    let mut downloaded = start_byte;

    while let Some(chunk_result) = stream.next().await {
        let chunk = chunk_result?;
        file.write_all(&chunk).await?;
        downloaded += chunk.len() as u64;

        // 发送进度（不要每个chunk都发，每64KB发一次减少开销）
        if downloaded % 65536 < chunk.len() as u64 {
            let _ = progress_tx.send(ProgressEvent::DownloadProgress {
                step_index: config.step_index,
                downloaded,
                total: total_size,
            }).await;
        }
    }

    // 最终进度
    let _ = progress_tx.send(ProgressEvent::DownloadProgress {
        step_index: config.step_index,
        downloaded,
        total: total_size,
    }).await;

    file.flush().await?;

    Ok(config.dest)
}
```

#### GitHub Release API查询
```rust
use serde::Deserialize;

#[derive(Deserialize)]
struct GithubRelease {
    tag_name: String,
    assets: Vec<GithubAsset>,
}

#[derive(Deserialize)]
struct GithubAsset {
    name: String,
    browser_download_url: String,
}

pub async fn get_github_release_url(
    repo: &str,
    asset_pattern: &str,
) -> Result<(String, String)> {
    let client = Client::new();
    let url = format!("https://api.github.com/repos/{}/releases/latest", repo);

    let response: GithubRelease = client
        .get(&url)
        .header("User-Agent", "vcp-installer")
        .send()
        .await?
        .json()
        .await?;

    // 在assets中查找匹配pattern的文件
    for asset in &response.assets {
        if asset.name.contains(asset_pattern) {
            return Ok((asset.browser_download_url.clone(), response.tag_name.clone()));
        }
    }

    anyhow::bail!("在 {} 的最新release中未找到匹配 '{}' 的文件", repo, asset_pattern);
}
```

#### Node.js LTS版本查询
```rust
use serde::Deserialize;

#[derive(Deserialize)]
struct NodeVersion {
    version: String,
    lts: serde_json::Value, // false 或 字符串（如 "Iron"）
}

pub async fn get_nodejs_lts_version() -> Result<String> {
    let client = Client::new();
    let versions: Vec<NodeVersion> = client
        .get("https://nodejs.org/dist/index.json")
        .send()
        .await?
        .json()
        .await?;

    // 找第一个lts不是false的版本
    for v in &versions {
        if v.lts.is_string() {
            return Ok(v.version.clone()); // 如 "v20.11.0"
        }
    }

    anyhow::bail!("无法获取Node.js LTS版本");
}
```

### 4.3 重试机制

所有网络请求应包裹在重试逻辑中：

```rust
/// 带重试的下载
pub async fn download_with_retry(
    config: DownloadConfig,
    progress_tx: mpsc::Sender<ProgressEvent>,
    max_retries: usize,
) -> Result<PathBuf> {
    let mut last_error = None;

    for attempt in 0..=max_retries {
        if attempt > 0 {
            let _ = progress_tx.send(ProgressEvent::Log(
                format!("下载重试 ({}/{}): {}", attempt, max_retries, config.url)
            )).await;
            tokio::time::sleep(std::time::Duration::from_secs(2 * attempt as u64)).await;
        }

        match download_file(
            DownloadConfig {
                url: config.url.clone(),
                dest: config.dest.clone(),
                step_index: config.step_index,
                resume: attempt > 0, // 重试时尝试续传
            },
            progress_tx.clone(),
        ).await {
            Ok(path) => return Ok(path),
            Err(e) => {
                last_error = Some(e);
            }
        }
    }

    Err(last_error.unwrap_or_else(|| anyhow::anyhow!("下载失败，已用尽重试次数")))
}
```

### 4.4 镜像URL处理

```rust
/// 将GitHub URL替换为镜像URL
pub fn apply_mirror(url: &str, mirror_prefix: &str) -> String {
    if mirror_prefix.is_empty() || mirror_prefix == "https://github.com/" {
        return url.to_string();
    }
    url.replace("https://github.com/", mirror_prefix)
}
```

```rust
pub enum ArchiveFormat {
    Zip,
    TarGz,
    SevenZip,
}

pub async fn extract(

    dest_dir: &Path,

/// 根据文件扩展名检测格式
```

### 5.2 实现要点

#### 格式检测
```rust
pub fn detect_format(path: &Path) -> Result<ArchiveFormat> {
    let name = path.file_name()
        .and_then(|n| n.to_str())
        .unwrap_or("")
        .to_lowercase();

    if name.ends_with(".zip") {
        Ok(ArchiveFormat::Zip)
    } else if name.ends_with(".tar.gz") || name.ends_with(".tgz") {
        Ok(ArchiveFormat::TarGz)
    } else if name.ends_with(".7z") || name.ends_with(".7z.exe") {
        Ok(ArchiveFormat::SevenZip)
    } else {
        anyhow::bail!("无法识别的压缩格式: {}", name)
    }
}
```

#### ZIP解压（Node.js用这个）
```rust
use zip::ZipArchive;
use std::fs;
use std::io;

fn extract_zip(archive_path: &Path, dest_dir: &Path) -> Result<PathBuf> {
    let file = fs::File::open(archive_path)?;
    let mut archive = ZipArchive::new(file)?;

    // 确保目标目录存在
    fs::create_dir_all(dest_dir)?;

    // 记录第一个目录名（通常是 node-v20.11.0-win-x64/）
    let mut root_dir: Option<String> = None;

    for i in 0..archive.len() {
        let mut file = archive.by_index(i)?;
        let outpath = dest_dir.join(file.mangled_name());

        // 记录根目录
        if root_dir.is_none() {
            if let Some(first_component) = file.mangled_name().components().next() {
                root_dir = Some(first_component.as_os_str().to_string_lossy().to_string());
            }
        }

        if file.name().ends_with('/') {
            fs::create_dir_all(&outpath)?;
        } else {
            if let Some(p) = outpath.parent() {
                fs::create_dir_all(p)?;
            }
            let mut outfile = fs::File::create(&outpath)?;
            io::copy(&mut file, &mut outfile)?;
        }

        // 设置文件权限（Unix）
        #[cfg(unix)]
        {
            use std::os::unix::fs::PermissionsExt;
            if let Some(mode) = file.unix_mode() {
                fs::set_permissions(&outpath, fs::Permissions::from_mode(mode))?;
            }
        }
    }

    Ok(dest_dir.join(root_dir.unwrap_or_default()))
}
```

#### tar.gz解压（Python用这个）
```rust
use flate2::read::GzDecoder;
use tar::Archive;

fn extract_tar_gz(archive_path: &Path, dest_dir: &Path) -> Result<PathBuf> {
    let file = fs::File::open(archive_path)?;
    let decoder = GzDecoder::new(file);
    let mut archive = Archive::new(decoder);

    fs::create_dir_all(dest_dir)?;
    archive.unpack(dest_dir)?;

    // 尝试找到解压后的根目录
    let mut root_dir = None;
    for entry in fs::read_dir(dest_dir)? {
        let entry = entry?;
        if entry.file_type()?.is_dir() {
            root_dir = Some(entry.path());
            break;
        }
    }

    Ok(root_dir.unwrap_or_else(|| dest_dir.to_path_buf()))
}
```

#### 7z解压（PortableGit用这个）
```rust
fn extract_7z(archive_path: &Path, dest_dir: &Path) -> Result<PathBuf> {
    fs::create_dir_all(dest_dir)?;

    // 使用 sevenz-rust crate
    sevenz_rust::decompress_file(archive_path, dest_dir)
        .map_err(|e| anyhow::anyhow!("7z解压失败: {}", e))?;

    Ok(dest_dir.to_path_buf())
}
```

#### 统一入口
```rust
pub async fn extract(
    archive_path: &Path,
    dest_dir: &Path,
) -> Result<PathBuf> {
    let format = detect_format(archive_path)?;

    // 解压是CPU密集操作，放到blocking线程
    let archive = archive_path.to_path_buf();
    let dest = dest_dir.to_path_buf();

    tokio::task::spawn_blocking(move || {
        match format {
            ArchiveFormat::Zip => extract_zip(&archive, &dest),
            ArchiveFormat::TarGz => extract_tar_gz(&archive, &dest),
            ArchiveFormat::SevenZip => extract_7z(&archive, &dest),
        }
    }).await?
}
```

### 5.3 7z降级方案

如果 `sevenz-rust` 解压失败，降级为调用系统命令：

```rust
fn extract_7z_fallback(archive_path: &Path, dest_dir: &Path) -> Result<PathBuf> {
    // PortableGit的7z.exe是自解压的，可以用 -o 参数指定目录
    // 或者尝试调用系统的7z.exe
    let output = std::process::Command::new(archive_path)
        .arg("-o")
        .arg(dest_dir)
        .arg("-y")  // 自动确认
        .output()
        .map_err(|e| anyhow::anyhow!("执行7z自解压失败: {}", e))?;

    if !output.status.success() {
        let stderr = String::from_utf8_lossy(&output.stderr);
        anyhow::bail!("7z自解压失败: {}", stderr);
    }

    Ok(dest_dir.to_path_buf())
}
```

## 七、验收标准

1. **`cargo build` 编译通过**
2. **单元测试**: 能下载一个小文件（如GitHub raw文件）并验证内容正确
3. **进度回报**: 下载过程中持续通过channel发送ProgressEvent
4. **断点续传**: 中断下载后重新调用，发送Range头从断点继续
5. **格式检测**: 正确识别.zip/.tar.gz/.7z文件
6. **ZIP解压**: 能解压Node.js的zip包
7. **tar.gz解压**: 能解压Python的tar.gz包
8. **7z解压**: 能解压PortableGit的7z.exe文件（或降级为自解压命令）
9. **GitHub API**: 能查询git-for-windows/git的最新release URL
10. **Node.js版本**: 能查询最新LTS版本号
