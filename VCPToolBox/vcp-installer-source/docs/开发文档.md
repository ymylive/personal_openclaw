## 1. 项目概述

### 1.1 目标
开发一个 Rust 编写的 VCP 全家桶一键部署工具，编译为单个 Windows exe。
内置TUI交互界面，自动完成依赖下载安装、代码克隆、配置引导。

### 1.2 定位
- **目标用户**: 首次部署 VCP 的 Windows 用户
- **交付形式**: PR 到 VCPToolBox 的 GitHub Release
- **核心原则**: 单exe零前置依赖、TUI配置表、全自动化

### 1.3 仓库现状
- VCPToolBox: https://github.com/lioensky/VCPToolBox (1.4k star / 227 fork / 零release)
- VCPChat: https://github.com/lioensky/VCPChat
- 许可证: CC BY-NC-SA 4.0

### 1.4 部署组件

| 组件 | 仓库/来源 | 作用 |
|------|-----------|------|
| VCPToolBox | github.com/lioensky/VCPToolBox | 后端核心 (Node.js + Python + Rust) |
| VCPChat | github.com/lioensky/VCPChat | 前端客户端 (Electron) |
| NewAPI | github.com/QuantumNous/new-api | API聚合管理 (Go, 提供预编译exe) |

## 2. 技术方案: Rust单exe + TUI + 在线下载Portable依赖

### 2.1 方案选型理由

| 对比项 | Rust exe | Batch+PowerShell | Go exe |
|--------|----------|------------------|--------|
| 额外依赖 | 零 | 零(但功能弱) | 零 |
| TUI界面 | ratatui(强大) | 无 | bubbletea(强大) |
| 体积 | ~3-5MB(不含运行时) | 几KB | ~5-8MB |
| 性能 | 极佳 | 一般 | 好 |
| 项目一致性 | ✅ VCP已有Rust组件 | ✅ 已有bat | 偏离技术栈 |
| 异步下载 | tokio+reqwest | 弱 | goroutine |
| 跨平台 | ✅ | ❌ Windows only | ✅ |

### 2.2 核心架构

```
vcp-installer.exe (单文件，~3-5MB)
    │
    ├─ TUI界面 (ratatui + crossterm)
    │   ├─ 欢迎页
    │   ├─ 环境检测页
    │   ├─ 组件选择页
    │   ├─ 配置表页 (填写API密钥等)
    │   ├─ 安装进度页 (多任务并行进度条)
    │   └─ 完成页
    │
    ├─ 依赖管理器
    │   ├─ 检测已安装: which crate
    │   ├─ 下载: reqwest (流式+进度)
    │   ├─ 解压: zip/flate2 crate
    │   └─ 环境变量: winreg crate
    │
    └─ 部署引擎
        ├─ git clone (调用下载的PortableGit)
        ├─ npm install (调用下载的Node.js)
        ├─ pip install (调用下载的Python)
        └─ config.env 生成
```

### 2.3 依赖下载策略: 在线下载Portable版本

exe本身不内嵌运行时（避免200MB+体积），运行时从官方源按需下载Portable版本。

| 依赖 | Portable来源 | 下载URL模式 | 解压后体积 |
|------|-------------|-------------|-----------|
| Git | Git for Windows Portable | `github.com/git-for-windows/git/releases/.../PortableGit-{v}-64-bit.7z.exe` | ~50MB |
| Node.js | 官方zip包 | `nodejs.org/dist/v{v}/node-v{v}-win-x64.zip` | ~30MB |
| Python | python-build-standalone | `github.com/indygreg/python-build-standalone/releases/...` | ~25MB |
| NewAPI | 官方exe | `github.com/QuantumNous/new-api/releases/.../new-api-{v}.exe` | ~58MB |

**关键**: 下载的Portable版本释放到安装目录的 `runtimes/` 子目录，完全不污染系统环境。

### 2.4 环境隔离方案

```
安装目录/
├── VCPToolBox/          ← git clone
├── VCPChat/             ← git clone
├── runtimes/            ← Portable运行时
│   ├── git/             ← PortableGit
│   ├── node/            ← Node.js portable
│   └── python/          ← python-build-standalone
├── new-api.exe          ← NewAPI (可选)
├── start-backend.bat    ← 生成的启动脚本(使用内置运行时)
├── start-frontend.bat   ← 生成的启动脚本
└── config-backup/       ← 配置备份
```

启动脚本使用内置运行时:
```batch
@echo off
set PATH=%~dp0runtimes\node;%~dp0runtimes\git\cmd;%~dp0runtimes\python;%PATH%
cd /d "%~dp0VCPToolBox"
node server.js
```

### 4.2 TUI事件循环核心

```rust
// 伪代码
enum AppState {
    Welcome,
    EnvCheck,
    ComponentSelect,
    ConfigForm,
    Installing(InstallProgress),
    Complete(InstallResult),
}

async fn run_app(terminal: &mut Terminal<CrosstermBackend>) -> Result<()> {
    let (tx, rx) = mpsc::channel(32); // 后台任务 → UI 的进度通道
    let mut app = App::new();
    
    loop {
        terminal.draw(|f| {
            match &app.state {
                AppState::Welcome => ui::welcome::render(f, &app),
                AppState::EnvCheck => ui::env_check::render(f, &app),
                AppState::ComponentSelect => ui::component_select::render(f, &app),
                AppState::ConfigForm => ui::config_form::render(f, &app),
                AppState::Installing(p) => ui::progress::render(f, p),
                AppState::Complete(r) => ui::complete::render(f, r),
            }
        })?;
        
        // 处理输入事件 + 后台进度
        tokio::select! {
            Some(event) = crossterm_events.next() => app.handle_input(event),
            Some(progress) = rx.recv() => app.update_progress(progress),
        }
        
        if app.should_quit { break; }
    }
    Ok(())
}
```

## 5. 安装流程详细设计

### 5.1 依赖下载 (downloader.rs)

```rust
// 流式下载 + 实时进度回报
async fn download_file(
    url: &str,
    dest: &Path,
    progress_tx: mpsc::Sender<ProgressEvent>,
) -> Result<()> {
    let response = reqwest::get(url).await?;
    let total_size = response.content_length().unwrap_or(0);
    let mut stream = response.bytes_stream();
    let mut file = tokio::fs::File::create(dest).await?;
    let mut downloaded: u64 = 0;
    
    while let Some(chunk) = stream.next().await {
        let chunk = chunk?;
        file.write_all(&chunk).await?;
        downloaded += chunk.len() as u64;
        progress_tx.send(ProgressEvent::Download {
            downloaded,
            total: total_size,
        }).await?;
    }
    Ok(())
}
```

### 5.2 Portable运行时管理

#### PortableGit
```
下载: PortableGit-{v}-64-bit.7z.exe
解压: sevenz-rust → runtimes/git/
验证: runtimes/git/cmd/git.exe --version
```

#### Node.js Portable
```
下载: node-v{v}-win-x64.zip
解压: zip crate → runtimes/node/
验证: runtimes/node/node.exe --version
附带: npm在runtimes/node/node_modules/npm/
```

#### Python Portable (python-build-standalone)
```
下载: cpython-{v}+{build}-x86_64-pc-windows-msvc-install_only.tar.gz
解压: flate2+tar → runtimes/python/
后处理: 
  1. 修改 python{ver}._pth 取消 #import site 注释
  2. 下载运行 get-pip.py
验证: runtimes/python/python.exe --version
```

### 5.3 部署步骤

```
步骤1: 下载Portable Git      → runtimes/git/
步骤2: 下载Portable Node.js   → runtimes/node/
步骤3: 下载Portable Python    → runtimes/python/ (如用户未安装)
步骤4: 设置临时PATH (指向runtimes/)
步骤5: git clone VCPToolBox
步骤6: cd VCPToolBox && npm install
步骤7: pip install -r requirements.txt
步骤8: 生成 config.env (基于配置表)
步骤9: git clone VCPChat (可选)
步骤10: cd VCPChat && npm install && pip install
步骤11: 下载NewAPI exe (可选)
步骤12: 生成启动脚本 (start-backend.bat / start-frontend.bat)
步骤13: 可选: 下载壁纸包/音频解码器 (VCPChat Releases)
```

## 7. 错误处理

| 错误场景 | 处理方式 |
|----------|----------|
| 网络不可达 | TUI提示检查网络，支持重试 |
| 下载中断 | 支持断点续传(HTTP Range) |
| 解压失败 | 删除损坏文件，提示重新下载 |
| git clone失败 | 提示切换镜像，支持重试 |
| npm install失败 | 显示错误日志，建议清缓存重试 |
| pip install失败 | 切换清华镜像重试 |
| 磁盘空间不足 | 安装前检测(需>=3GB) |
| 端口被占用 | 完成页提示检测端口 |
| 安装目录已存在 | 询问: 跳过/覆盖(git pull)/删除重来 |
| 路径含中文或空格 | 所有路径加引号处理 |
| 7z解压失败(PortableGit) | 降级为调用system命令解压 |
| Windows长路径 | 使用 \\\\?\\ 前缀绕过MAX_PATH |

## 12. 风险与注意事项

| 风险 | 影响 | 缓解 |
|------|------|------|
| PortableGit是7z格式 | 需要sevenz-rust crate | 备选: 调用系统7z或下载zip版 |
| python-build-standalone体积大 | 单文件~80MB压缩包 | 可选: 用官方embed包(更小但需要额外配置pip) |
| node_modules长路径 | Windows MAX_PATH=260 | 使用 \\\\?\\ 长路径前缀 |
